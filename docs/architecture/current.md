# OUTLOCAL Architecture

## Overview

OUTLOCAL is a 12-step automated cold outreach pipeline for UK local businesses. All components are Python async, sharing a single SQLite database.

## Data Flow

1. **Scrape** вҶ’ leads table (status: new)
2. **Enrich** вҶ’ leads table updated (status: enriched)
3. **Score** вҶ’ leads table updated (status: scored)
4. **Generate** вҶ’ emails table (status: draft)
5. **Send** вҶ’ emails table (status: sent)
6. **Follow-up** вҶ’ new emails linked to original
7. **Classify** вҶ’ replies table with classification
8. **Track** вҶ’ leads status updated (interested/converted/lost)

## Key Design Decisions

- **SQLite over PostgreSQL**: Zero-config, single-file, sufficient for solo founder scale (~300 emails/day)
- **Async-first**: aiosqlite, aiohttp, aiosmtplib вҖ” everything non-blocking
- **Free LLMs**: Groq вҶ’ OpenRouter вҶ’ Gemini failover chain, zero cost
- **No ORM**: Raw SQL with parameterised queries via aiosqlite вҖ” simpler, faster, no magic
- **Monolith**: Single FastAPI app with module boundaries, not microservices
